# Analysis: Homepage Internal Server Error on Vercel

## Root Cause (Most Likely)

**Missing Environment Variables in Vercel Dashboard**

The `.env.local` file is gitignored and NOT deployed to Vercel. The Supabase client in `lib/supabase.ts` uses non-null assertions (`!`) which will throw if env vars are undefined.

## Files Involved

| File              | Issue                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------- |
| `lib/supabase.ts` | Crashes if `NEXT_PUBLIC_SUPABASE_URL` or `NEXT_PUBLIC_SUPABASE_ANON_KEY` is undefined |
| `.env.local`      | Gitignored - not deployed to Vercel                                                   |
| `proxy.ts`        | Middleware in non-standard location (`app/middleware.ts` expected)                    |

## Required Fix

Set these environment variables in **Vercel Dashboard → Settings → Environment Variables**:

1. `NEXT_PUBLIC_SUPABASE_URL` = `https://phfuwunwgzkfzettekkh.supabase.co`
2. `NEXT_PUBLIC_SUPABASE_ANON_KEY` = (your anon key from .env.local)
3. `NEXT_SUPABASE_SERVICE_ROLE_KEY` = (your service role key from .env.local)

## Secondary Issues to Verify

1. **Middleware location**: `proxy.ts` should be `app/middleware.ts` for Next.js App Router
2. **PWA build**: `next.config.mjs` spawns git command which may fail in Vercel build env

## Verification Steps

1. Set env vars in Vercel dashboard
2. Redeploy
3. Check Vercel logs for specific error messages
