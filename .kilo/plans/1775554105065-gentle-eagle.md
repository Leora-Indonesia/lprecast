# Plan: Fix Stepper Checkmark Icon Not Showing

## Problem

The checkmark icon (`fa-check`) in the stepper component on `/app/(vendor)/vendor/register/page.tsx` does not render. Numbers render correctly, but when a step is completed, the checkmark icon is invisible.

## Root Cause

Font Awesome CSS is not loaded in the Next.js app. The code uses:

- `lucide-react` icons (installed: `"lucide-react": "^1.7.0"`)
- Font Awesome classes like `fa-solid fa-check` and `fa-arrow-left`

Font Awesome is only referenced via `<i className="fa-solid ...">` elements without the corresponding CSS being imported.

## Solution Options

### Option A: Replace Font Awesome with Lucide React (Recommended)

Since `lucide-react` is already installed and used throughout the project, replace the Font Awesome icon elements with equivalent Lucide icons.

**Changes to `app/(vendor)/vendor/register/page.tsx`:**

1. Import `Check` and `ArrowLeft` from `lucide-react`:

   ```tsx
   import {
     Building2,
     LogOut,
     Check,
     ArrowLeft,
     ArrowRight,
     PaperPlane,
     Spinner,
   } from "lucide-react"
   ```

2. Replace Font Awesome icons with Lucide equivalents:
   - `<i className="fa-solid fa-check text-xs" />` → `<Check className="h-3 w-3" />`
   - `<i className="fa-solid fa-arrow-left mr-2" />` → `<ArrowLeft className="mr-2 h-4 w-4" />`
   - `<i className="fa-solid fa-arrow-right text-xs" />` → `<ArrowRight className="h-3 w-3" />`
   - `<i className="fa-solid fa-paper-plane text-xs" />` → `<PaperPlane className="h-3 w-3" />`
   - `<i className="fa-solid fa-spinner fa-spin" />` → `<Spinner className="h-4 w-4 animate-spin" />`

### Option B: Add Font Awesome CSS via CDN

Add Font Awesome CSS link to `app/layout.tsx`:

```tsx
<link
  rel="stylesheet"
  href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css"
/>
```

## Recommended Approach

**Option A** is recommended because:

- The project already uses `lucide-react` (no new dependencies)
- Consistent icon library throughout the codebase
- Better integration with React (tree-shakeable, proper TypeScript types)
- Lucide icons follow Tailwind sizing conventions naturally

## Issues Found

### Issue 1: Border-bottom on all items

**Current (line 308-314):**

```tsx
className={`flex cursor-pointer items-center gap-3 border-b-2 px-8 py-4 transition ${
  isActive || isCompleted
    ? "border-primary bg-white text-primary"
    : "border-transparent text-gray-500 hover:text-gray-800"
} ${index < STEPS.length - 1 ? "border-r border-gray-100" : ""} ${
  isCompleted ? "cursor-pointer" : ""
}`}
```

**Problem:** `border-b-2` creates a bottom border on ALL items. User wants border-bottom ONLY on active item.

**Fix:** Move `border-b-2` into the conditional - only apply to active:

```tsx
className={`flex cursor-pointer items-center gap-3 px-8 py-4 transition ${
  isActive
    ? "border-b-2 border-primary bg-white text-primary"
    : "text-gray-500 hover:text-gray-800"
} ${isCompleted ? "bg-white text-primary" : ""} ${
  index < STEPS.length - 1 ? "border-r border-gray-100" : ""
} ${isCompleted ? "cursor-pointer" : ""}`}
```

### Issue 2: Vertical divider not showing

**Problem:** The conditional `border-primary` or `border-transparent` sets color for ALL borders (including right), overriding `border-gray-100`.

**CSS specificity issue:** When you have:

```tsx
isActive || isCompleted ? "border-primary" : "border-transparent"
```

This sets `border-color` for ALL sides, overriding the right border color.

**Fix:** Use granular border classes:

- `border-b-primary` or `border-b-transparent` for bottom only
- Keep right border separate with explicit color that won't be overridden

### Issue 3: Completed and Active have same styling

**Current:** Both completed and active show `border-primary` on bottom.

**Desired:**

- Active: Shows bottom border (indicates current position)
- Completed: No bottom border, shows filled circle with check
- Inactive: No bottom border, shows number

## Final Implementation

```tsx
<div
  key={index}
  id={`stepHeader${index}`}
  onClick={() => isCompleted && handleStepClick(index)}
  className={`flex cursor-pointer items-center gap-3 px-8 py-4 transition ${
    isActive
      ? "border-b-2 border-primary bg-white text-primary"
      : isCompleted
        ? "bg-white text-primary"
        : "bg-transparent text-gray-500 hover:text-gray-800"
  } ${index < STEPS.length - 1 ? "border-r border-gray-100" : ""} ${
    isCompleted ? "cursor-pointer" : ""
  }`}
>
```

**Changes:**

1. Remove `border-b-2` from base class
2. Only add `border-b-2 border-primary` when `isActive` is true
3. Keep `border-r border-gray-100` separate (won't be overridden now)
4. Completed gets `bg-white text-primary` but no border-bottom

## Additional Fix Needed: Right Border Color

**User feedback:** Right border should ALWAYS be `border-gray-100`, regardless of active/completed state.

**Current (line 314):**

```tsx
${
  index < STEPS.length - 1
    ? `border-r ${isActive ? "border-primary" : "border-gray-100"}`
    : ""
}
```

**Desired:** Right border should stay gray-100 for all states (active, completed, inactive).

**Fix:** Revert to simple conditional:

```tsx
${index < STEPS.length - 1 ? "border-r border-gray-100" : ""}
```

Right border is always `border-gray-100` regardless of state.

**Desired behavior:**

- Active: right border should be `border-primary`
- Completed: right border should be `border-gray-100` (matches overall separator)
- Inactive: right border should be `border-gray-100`

**Fix:** Make right border color conditional:

```tsx
${index < STEPS.length - 1 ? `border-r ${isActive ? "border-primary" : "border-gray-100"}` : ""}
```

Or using template literal:

```tsx
className={`... ${index < STEPS.length - 1 ? `border-r ${isActive ? "border-primary" : "border-gray-100"}` : ""} ...`}
```
