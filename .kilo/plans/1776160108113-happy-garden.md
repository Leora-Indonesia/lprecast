# Plan: Vendor Login Page

## Goal

Create a vendor login page at `/vendor/login` similar to the existing user login page at `app/(auth)/login/page.tsx`.

## Problem Found

**"Unauthorized" error on /vendor/login** — Two issues:

1. **Layout issue** (FIXED): Vendor login page at `app/(vendor)/vendor/login/` was wrapped by the protected `(vendor)` layout. Moved to `app/(auth)/vendor/login/` which has no auth check.

2. **Middleware issue** (ROOT CAUSE): The `proxy.ts` middleware blocks `/vendor/login` because:
   - It's NOT in `publicRoutes` array (only `/login` is, not `/vendor/login`)
   - It doesn't start with `/auth/`
   - So unauthenticated users → redirected to `/login`; authenticated non-vendor users → redirected to `/unauthorized`

Note: `/vendor/register` and `/vendor/register/success` are already in `publicRoutes`, so vendor registration works fine. We just need to add `/vendor/login` too.

## Implementation Steps

### 1. ✅ DONE — Vendor login page exists at `app/(auth)/vendor/login/page.tsx`

### 2. Update `proxy.ts` — Add `/vendor/login` to publicRoutes

```ts
const publicRoutes = [
  "/",
  "/login",
  "/vendor/login", // ADD THIS
  "/unauthorized",
  "/about",
  "/vendor/register",
  "/vendor/register/success",
  "/terms/vendor",
  "/terms/client",
  "/privacy",
]
```

This ensures the middleware allows unauthenticated access to `/vendor/login`.
