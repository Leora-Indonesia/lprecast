## 🐛 Bug Fix: project_summary VIEW

**Error:** `column p.project_progress does not exist`

**Penyebab:** VIEW `project_summary` mereferensi kolom yang tidak ada di tabel `projects`:

- `project_progress` - computed column
- `total_spk` - aggregated from vendor_spk
- `vendor_paid`, `vendor_outstanding` - aggregated from payments
- `customer_paid`, `customer_outstanding` - aggregated from customer_payment

**Solusi:** Simplify `project_summary` VIEW untuk hanya select dari `projects`:

```sql
CREATE VIEW project_summary AS
SELECT
    id,
    name,
    customer_name,
    contract_value,
    status,
    created_at,
    updated_at
FROM projects;
```

   - Hampir semua tabel ERP reference `users`

   - Supabase Auth terintegrasi dengan tabel `users`
   - Email/uid dari auth perlu dipertimbangkan

3. **Shared Master Data:**
   - `master_cities`, `master_provinces` mungkin dipakai keduanya

## Tantangan Utama

### 1. Foreign Key ke `users`

Banyak tabel LPrecast memiliki foreign key ke tabel `users`:

- `vendor_registrations.vendor_id` → users
- `vendor_registrations.reviewed_by` → users
- `vendor_profiles.user_id` → users
- `vendor_legal_documents.verified_by` → users
- `vendor_kpi_scores.vendor_id` → users
- dll.

**Solusi:** Buat tabel `users` terpisah di project LPrecast dengan struktur yang sama atau simplified.

### 2. Authentication Terpisah

User ingin auth terpisah total antara LPrecast dan ERP.

**Implikasi:**

- Perlu Supabase Auth project terpisah
- Tabel `auth.users` terpisah
- Session management terpisah
- User yang ada di LPrecast harus di-migrate atau re-register

### 3. Master Data Duplikasi

`master_provinces` dan `master_cities` perlu di-copy ke project baru.

**Solusi:** Copy data master ke project baru (sederhana, data statis).

### 4. View `user_profiles`

View ini bergantung pada tabel `users` dan `roles`. Perlu dibuat ulang di project baru dengan struktur yang sesuai.

## Rencana Migrasi

1. **Audit foreign key dependencies**  
   Identifikasi semua foreign key dari tabel LPrecast ke tabel non-LPrecast.

2. **Buat project Supabase baru** untuk LPrecast.

3. **Backup database existing**
   ```bash
   supabase db dump --db-url "[CONNECTION_STRING]" -f backup_roles.sql --role-only
   supabase db dump --db-url "[CONNECTION_STRING]" -f backup_schema.sql --schema-only
   supabase db dump --db-url "[CONNECTION_STRING]" -f backup_data.sql --use-copy --data-only
   ```

### Fase 2: Schema Migration (Hari 2-3)

4. **Buat SQL script untuk:**
   - Extract schema tabel LPrecast
   - Create tabel `users` baru untuk LPrecast (simplified)
   - Create view `user_profiles` baru
   - Handle foreign key constraints

5. **Import schema ke project baru**
   ```bash
   psql "$NEW_SUPABASE_DB_URL" < lprecast_schema.sql
   ```

### Fase 3: Data Migration (Hari 3-4)

6. **Export data dari tabel LPrecast**  
   Hanya data yang relevan untuk LPrecast.

7. **Transform dan import data**
   - Remap user IDs jika perlu

8. **Seed master data**
   - `master_provinces`
   - `master_cities`

### Fase 4: Application Update (Hari 4-5)

10. **Update authentication flow**
    - Callback URLs baru
    - Email templates baru

11. **Test semua fitur**
    - Vendor registration
    - Vendor profile management

```sql
-- ============================================
-- LPRECAST SCHEMA MIGRATION
-- untuk Project BARU (LPrecast)
-- ============================================

-- Note: Jalankan di project BARU (empty database)
-- Schema ini akan create tabel LPrecast tanpa data

-- CATATAN PENTING:
-- 1. master_provinces dan master_cities akan di-copy dari database lama
-- 2. users table di project BARU akan fresh (tanpa data dari ERP)
-- 3. ERP tetap mempertahankan tabel users yang original

-- ============================================
-- BACKUP DI DATABASE LAMA (ERP)
-- Jalankan INI DULU sebelum migrate
-- ============================================

-- Lihat bagian "Fase 1: Backup di Database Lama (ERP)" di atas

-- ============================================
-- MASTER DATA TABLES (Schema only, data akan di-copy)
-- ============================================

CREATE TABLE master_provinces (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    code TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE master_cities (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    code TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    province_id TEXT NOT NULL REFERENCES master_provinces(id),
    type TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- USERS TABLE (Fresh untuk LPrecast)
-- ============================================

CREATE TABLE users (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    email TEXT NOT NULL UNIQUE,
    nama TEXT NOT NULL,
    username TEXT NOT NULL UNIQUE,
    no_hp TEXT,
    jabatan TEXT,
    avatar_url TEXT,
    signature_image TEXT,
    stakeholder_type TEXT NOT NULL DEFAULT 'vendor',
    is_active BOOLEAN DEFAULT TRUE,
    last_login_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- View user_profiles untuk compatibility
CREATE VIEW user_profiles AS
SELECT
    id,
    email,
    nama,
    username,
    no_hp,
    jabatan,
    avatar_url,
    signature_image,
    stakeholder_type,
    is_active,
    last_login_at,
    created_at,
    updated_at
FROM users;

-- ============================================
-- VENDOR REGISTRATION & PROFILE TABLES
-- ============================================

CREATE TYPE vendor_registration_status AS ENUM (
    'draft', 'submitted', 'under_review', 'approved',
    'rejected', 'revision_requested'
);

CREATE TYPE vendor_profile_status AS ENUM (
    'active', 'suspended', 'blacklisted'
);

CREATE TYPE vendor_document_type AS ENUM (
    'ktp', 'npwp', 'nib', 'siup_sbu', 'company_profile'
);

CREATE TYPE document_verification_status AS ENUM (
    'pending', 'verified', 'rejected'
);

CREATE TYPE cost_inclusion_type AS ENUM (
    'mobilisasi_demobilisasi', 'penginapan_tukang',
    'biaya_pengiriman', 'biaya_langsir', 'instalasi', 'ppn'
);

CREATE TABLE vendor_registrations (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    vendor_id TEXT REFERENCES users(id),
    status vendor_registration_status DEFAULT 'draft',
    current_step INTEGER DEFAULT 1,
    legal_agreement BOOLEAN DEFAULT FALSE,
    submission_date TIMESTAMPTZ,
    reviewed_by TEXT REFERENCES users(id),
    reviewed_at TIMESTAMPTZ,
    approval_notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE vendor_profiles (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    user_id TEXT NOT NULL UNIQUE REFERENCES users(id),
    registration_id TEXT REFERENCES vendor_registrations(id),
    status vendor_profile_status DEFAULT 'active',
    preferred_vendor BOOLEAN DEFAULT FALSE,
    approved_by TEXT REFERENCES users(id),
    approved_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE vendor_company_info (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    registration_id TEXT NOT NULL REFERENCES vendor_registrations(id),
    nama_perusahaan TEXT NOT NULL,
    nama_pic TEXT NOT NULL,
    kontak_pic TEXT NOT NULL,
    email TEXT NOT NULL,
    website TEXT,
    instagram TEXT,
```
