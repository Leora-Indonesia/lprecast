# Implementasi Route Groups dan Middleware

## Tujuan

Mereorganisasi struktur routing aplikasi Next.js menggunakan route groups dan mengimplementasikan middleware redirect berdasarkan `stakeholder_type`.

## Analisis Current State

### Struktur Saat Ini

- Flat structure tanpa route groups: `app/vendor/dashboard`, `app/admin/dashboard`, `app/client/dashboard`, dll.
- Middleware berupa `proxy.ts` yang mengatur authentication
- Database memiliki table `user_profiles` dengan field `stakeholder_type` (enum: "internal" | "vendor" | "client")

### Link References yang Perlu Update

File `app/page.tsx` (homepage) memiliki link ke:

- `/login` (line 58, 83, 280)
- `/vendor/register` (line 53, 292)
