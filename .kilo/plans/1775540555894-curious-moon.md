# Fix Route Groups Path Conflict

## Problem

Route groups `(name)` are completely removed from the URL. The current structure creates conflicts:

- `app/(vendor)/dashboard/page.tsx` â†’ `/dashboard`
- `app/(admin)/dashboard/page.tsx` â†’ `/dashboard`
- `app/(client)/dashboard/page.tsx` â†’ `/dashboard`

All three resolve to the same URL path, causing Next.js routing errors.

## Solution

Restructure to include path segments inside route groups:

```
app/(vendor)/vendor/dashboard/page.tsx  â†’ /vendor/dashboard
app/(vendor)/vendor/profile/page.tsx    â†’ /vendor/profile
app/(admin)/admin/dashboard/page.tsx     â†’ /admin/dashboard
app/(client)/client/dashboard/page.tsx   â†’ /client/dashboard
```
