# Plan: Implement Vendor Approval Checklist System

## Context

The vendor approval process involves a comprehensive checklist of approximately 50 verification items during field visits. The current simple approve/reject workflow is insufficient - admins need a structured checklist interface to systematically verify each requirement before approving vendors.

## Objective

Replace the current simple approval workflow with a comprehensive checklist-based verification system that allows admins to check off each of ~50 items during the field visit and approval process.

## Current State Analysis

- ✅ Vendors list page exists at `/app/(admin)/admin/vendors/page.tsx`
- ✅ Shows pending vendors (submitted + under_review)
- ✅ Individual vendor detail page has basic approve/reject dialogs
- ✅ Server actions exist for approve/reject in `app/(admin)/admin/vendors/actions.ts`
- ❌ No checklist functionality exists
- ❌ No field visit verification workflow
- ❌ No structured approval process

## Key Requirements Clarification Needed

**CRITICAL**: Need detailed specification of the ~50 checklist items. What are the specific verification points for field visits? Examples might include:

- Facility infrastructure checks
- Equipment verification
- Document authenticity validation
- Operational capability assessments
- Compliance requirements
- Safety standards verification

## Implementation Approach

### Phase 1: Database Schema for Checklist System

1. Create `vendor_verification_checklists` table
2. Create `vendor_checklist_items` table (master checklist)
3. Create `vendor_checklist_responses` table (admin responses)
4. Add checklist status tracking to vendor_profiles

### Phase 2: Checklist Management Interface

1. Admin interface to view/manage master checklist items
2. Bulk import functionality for checklist items
3. Categorization system (facility, documents, operations, etc.)

### Phase 3: Vendor Verification Workflow

1. New checklist tab on vendor detail page
2. Interactive checklist with check/uncheck functionality
3. Progress tracking (completed/total items)
4. Photo upload capability for verification evidence
5. Notes/comments for each checklist item

### Phase 4: Approval Decision Logic

1. Automatic approval when all checklist items are verified
2. Conditional approval with outstanding items noted
3. Rejection with specific unverified items highlighted
4. Revision requests for missing/incorrect information

## Technical Considerations

### Checklist Item Types:

- Boolean (Yes/No) checks
- Text input for specifications
- Photo upload requirements

- Numeric thresholds (capacity, dimensions, etc.)

### Progress Calculation:

- Overall completion percentage
- Category-wise progress

### Data Storage:

- Master checklist items (reusable across vendors)

## Success Criteria

- Admins can systematically verify all ~50 checklist items
- Clear progress tracking and completion status
- Evidence collection (photos, notes) for each item
- Automated approval logic based on checklist completion
- Audit trail for compliance and accountability
- Mobile-friendly interface for field visits

- Allow checklist to be updated without affecting existing verifications
- Include fallback manual approval process
- Comprehensive testing of checklist logic and data integrity
