# Plan: Login Page untuk 4 User Roles

## Current State

Login page sudah ada di `app/(auth)/login/page.tsx` dengan:

- Email/password form menggunakan react-hook-form + zod
- Redirect berdasarkan `stakeholder_type`: internal → `/admin/dashboard`, vendor → `/vendor/dashboard`, client → `/client/dashboard`
- **Missing**: SPV role belum di-handle di redirect logic

## Problem

4 user roles yang perlu di-support:
| Role | stakeholder_type | Dashboard |
|------|------------------|------------|
| Admin | internal | /admin/dashboard |
| SPV | internal | /admin/dashboard |
| Vendor | vendor | /vendor/dashboard |
| Client | client | /client/dashboard |

Best practice dari Context7:

- Single login page untuk semua roles
- Redirect berdasarkan role setelah authentication berhasil
- Tidak perlu role selector di UI karena sudah ada di user data

## Recommended Approach

### 1. Update Redirect Logic (actions/auth.ts)

Tambahkan SPV handling:

```typescript
const stakeholderType = profile?.stakeholder_type

if (stakeholderType === "internal") {
  // Check if admin or SPV via separate flag or role field
  const isAdmin = profile?.role === "admin" // atau cek lain
  if (isAdmin) {
    redirect("/admin/dashboard")
  }
  redirect("/spv/dashboard") // SPV goes to separate dashboard
} else if (stakeholderType === "vendor") {
  redirect("/vendor/dashboard")
} else if (stakeholderType === "client") {
  redirect("/client/dashboard")
}
```

### 2. Enhance Login UI

Optional enhancements:

- Tambah logo/brand header di card
- Tambah "忘記密碼?" link (kalau needed)
- Feedback lebih jelas untuk invalid credentials
- Loading state yang jelas

### 3. Keep Design Simple

Single login page untuk semua roles sudah benar karena:

- User tahu kredensial mereka (tidak perlu Pilih role)
- Redirect otomatis berdasarkan stakeholder_type
- UI tetap clean dan professional
