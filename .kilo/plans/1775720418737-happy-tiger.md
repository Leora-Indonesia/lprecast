## Data Schema Reference

### vendor_registrations

- `id` (UUID), `vendor_id` (NULL), `status` ('submitted'|'approved'|'rejected')
- `reviewed_by`, `reviewed_at`, `approval_notes`

### vendor_company_info

- `email`, `nama_perusahaan`, `nama_pic`, `kontak_pic`

### vendor_profiles

- `user_id` (FK to users), `registration_id` (FK to vendor_registrations)
- `status` ('active'|'suspended'|'blacklisted'), `approved_by`, `approved_at`

### vendor_registration_status enum

```
'draft' | 'submitted' | 'under_review' | 'approved' | 'rejected' | 'revision_requested'
```

## Implementation Plan

### Step 1: Setup Email Infrastructure

**File: `lib/email.ts`**

- Nodemailer with SMTP credentials from cPanel
- Email templates for approved/rejected

**Environment Variables:**

```
SMTP_HOST=mail.domain.com
SMTP_PORT=587
SMTP_USER=noreply@domain.com
SMTP_PASS=password
SMTP_FROM="LPrecast Vendor Portal <noreply@domain.com>"
```

---

### Step 2: Create Admin Approval Server Actions

**File: `app/(admin)/admin/vendors/actions.ts`** (new file)

- `approveVendor(registrationId: string, adminId: string)`
  1. Get registration + company_info from DB
  2. Create user via Supabase Admin API with email from company_info
  3. Generate password reset link
  4. Create vendor_profiles record
  5. Update vendor_registrations.vendor_id + status='approved' + reviewed_at + reviewed_by
  6. Send approved email with password reset link
- `rejectVendor(registrationId: string, adminId: string, notes?: string)`
  1. Update vendor_registrations status='rejected' + reviewed_at + reviewed_by + approval_notes
  2. Send rejected email

---
