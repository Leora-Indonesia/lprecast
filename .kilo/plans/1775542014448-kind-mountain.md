# Vendor Registration Form - Implementation Plan

## Overview

Convert HTML wireframe (`public/wireframes/vendor/register.html`) to a Next.js page component (`app/(vendor)/vendor/register/page.tsx`) using shadcn/ui, React Hook Form, Zod validation, and TypeScript.

## Database Schema Mapping

### Related Tables:

| Table                      | Fields                                                                                            |
| -------------------------- | ------------------------------------------------------------------------------------------------- |
| `vendor_registrations`     | Main registration record (current_step, status, legal_agreement)                                  |
| `vendor_company_info`      | nama_perusahaan, email, nama_pic, kontak_pic, website, instagram, facebook, linkedin              |
| `vendor_contacts`          | nama, no_hp, jabatan, is_primary, sequence                                                        |
| `vendor_legal_documents`   | document_type (ktp/npwp/nib/siup_sbu/company_profile), document_number, file_path                 |
| `vendor_bank_accounts`     | bank_name, account_number, account_holder_name, is_primary                                        |
| `vendor_factory_addresses` | address, province, kabupaten, kecamatan, postal_code, map_url                                     |
| `vendor_products`          | name, satuan, price, dimensions, material, finishing, weight_kg, lead_time_days, moq, description |
| `vendor_delivery_areas`    | provinsi, kabupaten                                                                               |
| `vendor_cost_inclusions`   | inclusion_type, is_included                                                                       |
| `vendor_additional_costs`  | description, amount, unit                                                                         |

### Enums Used:

- `vendor_document_type`: 'ktp' | 'npwp' | 'nib' | 'siup_sbu' | 'company_profile'
- `cost_inclusion_type`: 'mobilisasi_demobilisasi' | 'penginapan_tukang' | 'biaya_pengiriman' | 'biaya_langsir' | 'instalasi' | 'ppn'

### Step 2: Create Zod Schemas

File: `lib/validations/vendor-registration.ts`

```typescript
// Step 1: Company Info Schema
const companyInfoSchema = z.object({
  namaPerusahaan: z.string().min(1, "Nama perusahaan wajib diisi"),
  email: z.string().email("Format email tidak valid"),
  namaPIC: z.string().min(1, "Nama PIC wajib diisi"),
  kontakPIC: z.string().min(1, "Kontak PIC wajib diisi"),
  // ... contacts array, social media
})

// Step 2: Legal Documents Schema
const legalDocsSchema = z.object({
  ktpFile: z.any().refine(/* file validation */),
  npwpNomor: z.string().optional(),
  npwpFile: z.any().optional(),
  // ... other documents
})

// Step 3: Operational Schema
const operationalSchema = z.object({
  bankNama: z.string().min(1, "Nama bank wajib diisi"),
  bankNomor: z.string().min(1, "Nomor rekening wajib diisi"),
  // ... factory address, products array, delivery areas, costs
})

// Full schema combining all steps
const vendorRegistrationSchema = z.object({
  companyInfo: companyInfoSchema,
  legalDocs: legalDocsSchema,
  operational: operationalSchema,
  legalAgreement: z.boolean().refine((val) => val === true),
})
```

### Step 4: State Management

Use React state for:

- `currentStep`: number (0-3)
- `products`: Product[] array
- `deliveryAreas`: DeliveryArea[] array
- Form data persisted via RHF

### Step 5: File Upload Handling

For document uploads (KTP, NPWP, NIB, SIUP, Company Profile):

- Use `<input type="file" />` with accept=".pdf,.jpg,.jpeg,.png"
- Store files in local state temporarily
- Upload to Supabase Storage on final submit

### Step 6: Form Submission Flow

On final submit:

1. Validate all steps
2. Create auth user (if new)
3. Insert `vendor_registrations` record
4. Insert related records (company_info, contacts, documents, bank_accounts, factory_addresses, products, delivery_areas, cost_inclusions, additional_costs)
5. Redirect to success page

## Key shadcn/ui Patterns to Use

### Form Field Pattern:

```tsx
<FormField
  control={form.control}
  name="namaPerusahaan"
  render={({ field }) => (
    <FormItem>
      <FormLabel>Nama Perusahaan*</FormLabel>
      <FormControl>
        <Input placeholder="PT. Maju Jaya" {...field} />
      </FormControl>
      <FormMessage />
    </FormItem>
  )}
/>
```

### Select Pattern:

```tsx
<Select onValueChange={field.onChange} defaultValue={field.value}>
  <SelectTrigger>
    <SelectValue placeholder="Pilih..." />
  </SelectTrigger>
  <SelectContent>
    <SelectItem value="meter">meter</SelectItem>
    <SelectItem value="lembar">lembar</SelectItem>
  </SelectContent>
</Select>
```

## Decisions Made

1. **File Uploads**: Upload on final submit - files stored in local state until form submission
2. **Draft Save**: Auto-save drafts todatabase for session persistence
3. **Data Layer**: Use Next.js Server Actions for data mutation

## Server Actions Structure

```
app/(vendor)/vendor/register/
└── actions.ts
    ├── saveDraft() - Auto-save form progress
    ├── loadDraft() - Load existing draft
    └── submitRegistration() - Final submission with file uploads
```

## Additional Features

- Auto-save indicator ("Perubahan Otomatis Tersimpan")
- Draft resumtion when user returns to page
- Progress tracking via `current_step` field
